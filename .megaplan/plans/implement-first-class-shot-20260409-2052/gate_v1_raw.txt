        # Implementation Plan: First-Class Shot Selection & Structured Timeline Context

## Overview

The video editor's AI timeline agent currently receives a flat clip list with no shot-group awareness. The frontend has clip-level selection but no way to select an entire shot group with one click. The `describeTimeline()` output lists clips without grouping them by shot. The "duplicate/edit/place-after-selected" workflow exists in the system prompt as instructions, but lacks structured shot context and reliable "same-shot" semantics on the agent side.

**Current state:**
- `describeTimeline()` (`supabase/functions/ai-timeline-agent/tools/timeline.ts:97-112`) produces flat clip lines with no shot grouping
- `SelectedClipPayload` (`types.ts:82-88`) has no `shot_id` field
- `buildSelectedClipsPrompt()` (`prompts.ts:40-68`) enriches selected clips with timeline line matching but no shot context
- Frontend `useTimelineSelection` supports multi-select via shift/toggle but has no shot-group-level selection
- `TimelineCanvas` renders shot groups visually (`shotGroups` prop) but clicking the shot group label/background only opens a context menu
- `useShotGroups` (`hooks/useShotGroups.ts`) derives `ShotGroup` objects with `shotId`, `clipIds`, `shotName` from pinned groups
- The agent's `duplicate_generation` tool resolves shot via `findShotForGenerations()` but the agent has no structured way to know which clips belong to which shot

**Goal:** Wire shot-group awareness end-to-end: single-click shot selection in the timeline UI, enriched agent payloads with shot metadata, reliable same-shot semantics in agent tools, and tests.

## Phase 1: Agent-Side Shot Context (Backend)

### Step 1: Add shot group info to `describeTimeline()` (`supabase/functions/ai-timeline-agent/tools/timeline.ts`)
**Scope:** Small

1. **Modify** `describeTimeline()` (line 97) to accept `pinnedShotGroups` from `TimelineConfig` and output a "Shot groups:" section after clips. Each shot group line should list: shot name, shot ID, track, clip IDs, and mode.
   ```
   Shot groups:
   - "Hero shots" (shot=abc track=V1 clips=[clip-0,clip-1,clip-2] mode=images)
   - "Transition" (shot=def track=V1 clips=[clip-3] mode=video)
   ```
2. **Update** `viewTimeline()` (line 114) — it already receives `config` which contains `pinnedShotGroups`, so no signature change needed. Just pass `config.pinnedShotGroups` into the updated `describeTimeline()`.
3. **Update** `formatClipLine()` (line 57) to include `shot=<shotName>` when a clip belongs to a pinned shot group, so each clip line is self-contained.

### Step 2: Add `shot_id` to `SelectedClipPayload` and enrich `buildSelectedClipsPrompt()` (`supabase/functions/ai-timeline-agent/types.ts`, `prompts.ts`)
**Scope:** Small

1. **Add** optional `shot_id?: string` and `shot_name?: string` fields to `SelectedClipPayload` (types.ts:82).
2. **Update** `buildSelectedClipsPrompt()` (prompts.ts:49-64) to include `shot=<shot_name>` in the selected clip lines when present.
3. **Update** the system prompt in `buildTimelineAgentSystemPrompt()` (prompts.ts:154-158) to reference shot context: "When the user selects clips from the same shot group, prefer operations within that shot."

### Step 3: Enrich agent invocation with shot context (`supabase/functions/ai-timeline-agent/loop.ts`)
**Scope:** Small

1. **After** `resolveClipGenerationIds()` (loop.ts:381-385), look up shot group membership for selected clips from `timelineState.config.pinnedShotGroups`. For each selected clip, if it belongs to a pinned group, attach `shot_id` and `shot_name` (via a DB lookup or from the config).
2. **Update** the `selectedClips` normalization in `index.ts` to pass through `shot_id`/`shot_name` if provided by the frontend.

### Step 4: Tests for agent-side shot context (`supabase/functions/ai-timeline-agent/tools/timeline.test.ts`, `loop.test.ts`)
**Scope:** Medium

1. **Create** `supabase/functions/ai-timeline-agent/tools/timeline.test.ts` with tests for:
   - `describeTimeline()` includes "Shot groups:" section when `pinnedShotGroups` are present
   - `describeTimeline()` omits "Shot groups:" when no pinned groups exist
   - `formatClipLine()` includes `shot=...` for clips in a pinned group
2. **Add** tests to existing `loop.test.ts` for:
   - `buildSelectedClipsPrompt()` includes shot info when `shot_id`/`shot_name` present
   - Selected clips without shot info still render correctly (backwards compat)

## Phase 2: Frontend Shot-Group Selection

### Step 5: Add shot-group click-to-select in `TimelineCanvas` (`src/tools/video-editor/components/TimelineEditor/TimelineCanvas.tsx`)
**Scope:** Medium

1. **Add** an `onShotGroupSelect?: (group: { shotId: string; clipIds: string[] }) => void` prop to `TimelineCanvasProps` (line 114 area).
2. **In** the shot group rendering section (around line 1065 where `data-action-id="shot-group-label"` is rendered), add an `onClick` handler on the shot group label/background that calls `onShotGroupSelect` with the group's `clipIds`.
3. **Wire** in `TimelineEditor.tsx` (around line 848): pass `onShotGroupSelect` that calls `selectClips(group.clipIds)` from the existing selection hook. This selects all clips in the shot group.

### Step 6: Pass shot context from frontend to agent payload (`src/tools/video-editor/hooks/useAgentSession.ts`, `useSelectedMediaClips.ts`)
**Scope:** Medium

1. **Update** `useSelectedMediaClips()` (`hooks/useSelectedMediaClips.ts:29`) to resolve shot group membership for each selected clip. Use the `pinnedShotGroups` from timeline config (available via `resolvedConfig`). Add `shotId?: string` and `shotName?: string` to `SelectedMediaClip`.
2. **Update** the agent invocation in `useAgentSession.ts` (line 276) to include `shot_id` and `shot_name` in the `selected_clips` payload when available.

### Step 7: Duplicate/edit/place-after-selected with shot awareness (`supabase/functions/ai-timeline-agent/tools/duplicate-generation.ts`)
**Scope:** Small

1. **When** `selectedClips` include `shot_id`, use it directly in `executeDuplicateGeneration()` (line 38) instead of the fallback `findShotForGenerations()` lookup. This makes same-shot semantics reliable — the frontend knows which shot group the clip belongs to.
2. **Update** the duplicate result message (line 68) to include the shot name when known, so the agent can reference it in follow-up operations.

### Step 8: Tests for frontend changes and duplicate workflow
**Scope:** Medium

1. **Add** tests to `useAgentSession.test.tsx` verifying that `shot_id` and `shot_name` are included in the `selected_clips` payload when clips belong to a pinned shot group.
2. **Add** a test case for the duplicate-generation tool that verifies `shot_id` from `selectedClips` is used directly (skipping the `findShotForGenerations` DB call).
3. **Add** a `TimelineCanvas` test (in existing `TimelineCanvas.test.tsx`) that verifies clicking a shot group label calls `onShotGroupSelect` with the correct clip IDs.
